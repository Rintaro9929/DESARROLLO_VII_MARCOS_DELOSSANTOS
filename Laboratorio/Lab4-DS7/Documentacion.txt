# 📌 Descripción del proyecto

Este laboratorio consiste en la implementación del estándar **PSR-4** en PHP utilizando **Composer** para automatizar la carga de clases.

El objetivo principal es eliminar el uso manual de instrucciones como:

```php
include
require
include_once
require_once
```

y reemplazarlas por un sistema de carga automática profesional mediante:

```php
vendor/autoload.php
```

---

# 🎯 Objetivos

* Comprender el funcionamiento del estándar **PSR-4**
* Configurar correctamente el archivo `composer.json`
* Implementar namespaces en una aplicación PHP

# ⚙️ Configuración de Composer

## Archivo `composer.json`

```json
{
  "name": "lab4-ds7/autoload-psr4",
  "version": "1.0.0",
  "autoload": {
    "psr-4": {
      "App\\": "app/"
    }
  }
}
```

# 🧠 Conclusiones técnicas

## 1. Mantenibilidad del código

La implementación de PSR-4 facilita la incorporación de nuevas clases sin modificar archivos globales de configuración. Esto mejora significativamente el mantenimiento del sistema.

---

## 2. Optimización mediante carga bajo demanda

Composer carga únicamente las clases necesarias durante la ejecución, optimizando el consumo de memoria y mejorando el rendimiento general.

---

## 3. Estandarización profesional

El uso del estándar PSR-4 permite desarrollar aplicaciones siguiendo buenas prácticas reconocidas en la industria, facilitando el trabajo colaborativo.

---

## 4. Escalabilidad

La organización basada en namespaces permite que el proyecto crezca ordenadamente sin perder estructura.

---

## 5. Reducción de errores

Se eliminan errores frecuentes asociados al uso incorrecto de rutas en `include` y `require`.
